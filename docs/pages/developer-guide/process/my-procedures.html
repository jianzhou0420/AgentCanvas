My Solo Development Procedures

1. Docs Maintenance

1.1 Core Docs

The project must maintain these 6 docs in docs-site/docs/:

Doc Purpose Update when
blueprint.md Problem → Solution → Principles → Components → Features Goals or direction shift
architecture.md System diagram, data flow, tech stack Structure changes
glossary.md Domain terms and definitions New concepts introduced
decisions/ ADRs — key choices and rationale, per-field Any non-trivial technical decision
roadmap.md Done / In Progress / Planned / Deferred Features ship or priorities change
my-procedures.md This file — how I work Process evolves

1.2 Feature Docs

Feature design docs in docs-site/docs/design-docs/:

Doc Purpose Update when
canvas-system.md Canvas system — composite nodes, nesting, navigation, node specs Canvas architecture changes
graph-system.md Graph data model — GraphDefinition, file management, API, serialization Graph model or API changes
graph-executor.md Dataflow execution engine — node firing, iteration, state Executor behavior changes
wire-types.md Wire type system — type catalog, serialization, canvas colors Wire types added or changed
nodesets.md NodeSet system — local vs server, lifecycle, examples NodeSet API changes
plugin-servers.md Plugin server architecture — BaseServer, ServerApp, manifest protocol Server system changes
state-containers.md Dual-wire state system — visible containers, reducers, checkpointing State system or container types change
(add more as features grow)

1.3 Implementation Docs

Technical reference docs in docs-site/docs/concepts/, tutorial/, recipes/, api-reference/:

Doc Purpose Update when
graph-expressible-agents.md Agent forms — DAG, agent loop, mixed composition Agent form or IterIn/IterOut changes
customizable-node-system.md Node system — base classes, ports, auto-discovery Node system API changes
isolated-runtime-environments.md Runtime isolation — server mode, AutoServerApp, proxy nodes Runtime or isolation changes
nested-graph-system.md Nested graphs — subgraph, PortIn/PortOut, flatten Graph nesting changes
graph-execution-engine.md Dataflow executor — firing rules, state containers Executor behavior changes
real-time-observability.md Observability — WebSocket, output viewers, streaming Observability system changes

1.4 Doc-Site — Single Source of Truth

  • Live documentation site (docs-site/, MkDocs Material) hosts all project docs
  • Serve locally: bash docs-site/run_dev.sh (live reload on save)

1.5 Change Tracking

  • Every doc has a last updated: timestamp and a ## Changelog section at the bottom
  • When a decision ripples across docs: write the ADR first (/adr), then propagate to each affected doc
  • Each affected doc gets its own changelog entry referencing the ADR (e.g. "see ADR-platform-001")
