CMA Policy on AgentCanvas — inference contract + policy_cma port

The Cross-Modal Attention policy is the classic trained VLN-CE baseline (Krantz et al., VLN-CE): a recurrent network attends across instruction tokens and RGB-D features and emits one of four discrete actions per step. This is a policy page (template §4): there is no prompt pipeline to analyse — §1 documents the inference contract of the released checkpoint, and the port wraps the whole policy as a single batched forward node, policy_cma__forward, with the RNN state carried on wires.

1. Upstream inference contract (Krantz et al., VLN-CE)

The VLN-CE evaluation loop drives the trained policy with four recurrent inputs per step:

  1. Inputs: observations (RGB, depth, tokenized instruction), rnn_states (GRU hidden, zeros at episode start), prev_actions (int64, 0 at start), not_done_masks (0 on the first step of an episode, 1 after — this is how the recurrent state resets without re-allocating).
  2. Forward: instruction and visual encoders → cross-modal attention → GRU → action distribution; eval uses the distribution mode (deterministic), not a sample.
  3. Outputs: a discrete action in {0 STOP, 1 FORWARD, 2 LEFT, 3 RIGHT} + the next rnn_states.
  4. Episode protocol: STOP (or budget) ends the episode; success is position-based (≤ 3 m), computed by the env.

2.3 State & memory

The recurrent state is explicit wire data: hidden_out → iterOut (persist) → iterIn → hidden_in. This replaces the prior per-instance ctx.policy / ctx.rnn_states side channel and is what makes the batched tier sound — the policy subprocess holds only the checkpoint (module-level _POLICY_BUNDLE + load lock), so K parallel workers can rendezvous in the BatchedInferenceServer (batched=True, batch_dim="raw_obs") for one stacked forward per step with zero cross-worker state bleed (ADR-028 PC-3).

2.4 Boundary contract

Server-mode in the dedicated ac-vlnce env (Python 3.8 + habitat-sim 0.1.7). The framework import boundary holds by construction: BaseVLNPolicy / RecurrentPolicy / CMAPolicy are defined inside the lazy load function, so importing the nodeset never imports vlnce_baselines or habitat. Checkpoint path: data/habitat/checkpoints/CMA_PM_DA_Aug.pth (resolved from repo root; VLNCE_ROOT env override supported).

6. What this nodeset is NOT

  • Not a reasoning method. No prompts, no LLM — a trained checkpoint behind one forward port.
  • Not the VLA stack. Manipulation policies (Pi0/SmolVLA/DP/…) live in policy_vla; this is the VLN-CE navigation baseline.
  • Not env logic. Episode control, metrics, and the discrete-action execution live in env_habitat.
