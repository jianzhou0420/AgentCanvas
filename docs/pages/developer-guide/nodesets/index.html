NodeSet Implementations

Implementation reference for domain-specific NodeSets in workspace/nodesets/. Each page documents a single NodeSet: its purpose, canvas nodes, port schemas, and usage patterns.

For the NodeSet system design — concept, API, three deployment modes, decision guide, cookbook, file map — see NodeSets (design doc). Companion: Base Canvas Node (the class your tools subclass).

NodeSets are grouped into four families, mirrored in the sidebar: env (simulators / benchmark environments), method (agent + policy implementations), model (foundation-model wrappers — generic vision / VLM primitives), and common (reusable building blocks — tools).
