SAM NodeSet

The SAM NodeSet provides visual segmentation and object tracking tools powered by Segment Anything Model (Meta, 2023) and its successors. Agents can segment objects referenced in navigation instructions, automatically discover objects in scenes, and track landmarks across video frames.

Three backends are supported: - SAM — point/box/auto-mask segmentation - SAM2 — video object tracking with memory attention - SAM3 — native text-prompted segmentation and 3D-aware masks (planned)

1. Overview

1.1 Purpose

Vision-language navigation (VLN) agents must understand the spatial layout of environments and locate objects mentioned in instructions. The SAM NodeSet provides canvas nodes for:

  • Segment-by-point: Click or provide pixel coordinates to isolate an object
  • Segment-by-box: Define a bounding box to segment its primary content
  • Auto-segmentation: Discover all objects in a scene without prompts
  • Text-prompted segmentation: Search for objects by natural language (SAM3 backend)
  • Video tracking: Track identified landmarks across frames (SAM2+ backends)

All six tools are always available on the canvas. Tools that require unsupported backend features return a descriptive error at execute-time.

1.2 Backend Architecture

The nodeset uses an abstract backend pattern: a common interface (SamBackend) with concrete implementations for each model version. This enables:

  • Seamless version switching via environment variables — change SAM_VERSION=sam2 and the same graph works with a different backend
  • Feature detection — tools check backend.supports_tracking / backend.supports_text_prompt to return errors for unsupported operations
  • Lazy initialization — the backend is loaded only when the nodeset is activated
Backend Status Point/Box/Auto Video Tracking Text Prompts Device Support
SAM Installed ✓ — — CUDA / CPU
SAM2 Mock ✓ ✓ — CUDA / CPU (when available)
SAM3 Mock ✓ ✓ ✓ CUDA / CPU (when released)

1.3 Backend Selection Strategy

The nodeset resolves the active backend at initialize-time based on environment variables:

# Environment variables (defaults shown)
SAM_VERSION = "sam2"              # sam | sam2 | sam3
SAM_MODEL_CFG = "default"         # checkpoint path or config name
SAM_DEVICE = "cuda:0"             # cuda:N or cpu

If the requested version is unavailable (e.g., SAM2 mock backend), tools will return errors like:

{
  "error": "sam2 does not support tracking. Use sam2 or sam3."
}

2. Backend Selection

2.1 Environment Variables

SAM_VERSION — selects which backend to use

  • sam — Original SAM (2023). Fastest inference, smallest memory footprint. No tracking or text prompts.
  • sam2 — Video SAM (2024). Slower but supports object tracking. Requires Python 3.10+.
  • sam3 — Upcoming. Planned to add native text prompts and 3D-aware masks.

Default: sam2

SAM_MODEL_CFG — path to model checkpoint or config name

  • "default" — uses data/habitat/checkpoints/sam/sam_vit_b.pth (SAM v1 only)
  • Path to .pth file — e.g. data/habitat/checkpoints/sam/sam_vit_l.pth
  • For SAM2: "default" resolves to the meta-provided checkpoint

Default: "default"

SAM_DEVICE — where to run the model

  • cuda:0 — first GPU (recommended for performance)
  • cuda:1, cuda:2 — other GPUs
  • cpu — CPU inference (slow, useful for testing or single-object scenarios)

Default: cuda:0

2.3 Checkpoint Models

SAM v1 supports three Vision Transformer sizes:

  • sam_vit_b.pth — 91M params, fastest (recommended)
  • sam_vit_l.pth — 308M params, slower, better quality
  • sam_vit_h.pth — 632M params, slowest, highest quality

The nodeset auto-detects model size from the checkpoint filename (_vit_b, _vit_l, _vit_h).

3. Canvas Nodes

All six nodes are registered when the SAM NodeSet is loaded. Tools that reference unsupported features (e.g., tracking on SAM v1) return a descriptive error.

Node Type Display Name Category Input Ports Output Ports Description
model_sam__segment_point SAM: Segment Point tool image_b64 (TEXT), points (TEXT), labels (TEXT), multimask (TEXT, opt) result (TEXT) Segment objects at specified point(s). Provide foreground points (label=1) on the target and background points (label=0) to exclude regions. Returns ranked candidate masks.
model_sam__segment_box SAM: Segment Box tool image_b64 (TEXT), box (TEXT) result (TEXT) Segment the primary object within a bounding box. Useful when the target's approximate location is known from detection or instruction.
model_sam__auto_mask SAM: Auto Mask tool image_b64 (TEXT), points_per_side (TEXT, opt), min_mask_area (TEXT, opt) result (TEXT) Automatically segment all objects in the image. Returns masks sorted by area. Useful for scene understanding and discovering objects the agent can interact with.
model_sam__segment_text [Mock] SAM: Segment Text tool image_b64 (TEXT), text (TEXT), threshold (TEXT, opt) result (TEXT) [MOCK] Segment objects matching a natural language description (e.g. 'brown wooden chair'). Returns random placeholder masks. Real implementation requires SAM3 backend (not yet available).
model_sam__track_init [Mock] SAM: Track Init tool image_b64 (TEXT), mask_index (TEXT), points (TEXT, opt) result (TEXT) [MOCK] Start tracking a segmented object across subsequent frames. Returns placeholder track_id. Real implementation requires SAM2 or SAM3 backend (not yet installed).
model_sam__track_propagate [Mock] SAM: Track Propagate tool image_b64 (TEXT), track_ids (TEXT) result (TEXT) [MOCK] Feed a new frame to the tracker and get updated bboxes/scores for all tracked objects. Returns random placeholder data. Real implementation requires SAM2 or SAM3 backend (not yet installed).

3.1 Segment Point

Node Type: model_sam__segment_point

Segment objects by providing one or more pixel coordinates. Background points (label=0) help exclude unwanted regions.

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image - points (TEXT) — JSON array of [x, y] pixel coordinates, e.g. [[100, 200], [150, 180]] - labels (TEXT) — JSON array of per-point labels: 1 = foreground (on target), 0 = background (exclude) - multimask (TEXT, optional) — true returns 3 candidate masks ranked by score; false returns 1. Default: true

Output Ports: - result (TEXT) — JSON object with segmentation results

Result Schema: 

{
  "masks": [
    {
      "mask_index": 0,
      "bbox": [x1, y1, x2, y2],
      "score": 0.95,
      "area": 5000,
      "mask_b64": "base64-encoded-png"
    }
  ],
  "count": 3,
  "prompt_type": "point"
}

3.2 Segment Box

Node Type: model_sam__segment_box

Segment the primary object within a bounding box. Faster and more direct when the target location is approximately known.

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image - box (TEXT) — JSON [x1, y1, x2, y2] bounding box in pixels, e.g. [100, 150, 250, 300]

Output Ports: - result (TEXT) — JSON object with segmentation results

Result Schema: 

{
  "masks": [
    {
      "mask_index": 0,
      "bbox": [x1, y1, x2, y2],
      "score": 0.92,
      "area": 4500,
      "mask_b64": "base64-encoded-png"
    }
  ],
  "count": 1,
  "prompt_type": "box"
}

3.3 Auto Mask

Node Type: model_sam__auto_mask

Automatically segment all salient objects in an image without any prompt. Returns masks sorted by area (largest first).

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image - points_per_side (TEXT, optional) — Grid density for point sampling. Higher = more masks, slower. Default: 32 - min_mask_area (TEXT, optional) — Discard masks smaller than this pixel area. Default: 100

Output Ports: - result (TEXT) — JSON object with segmentation results

Result Schema: 

{
  "masks": [
    {
      "mask_index": 0,
      "bbox": [x1, y1, x2, y2],
      "score": 0.88,
      "area": 25000,
      "stability_score": 0.95,
      "mask_b64": "base64-encoded-png"
    }
  ],
  "count": 12
}

3.4 Segment Text (SAM3, Mock)

Node Type: model_sam__segment_text

STATUS: Mock backend — returns random placeholder data. Real implementation awaits SAM3 release.

Segment objects matching a natural language description, e.g. "brown wooden chair" or "person wearing red shirt".

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image - text (TEXT) — Natural language description of the target object(s), e.g. "wooden dining table" - threshold (TEXT, optional) — Confidence threshold (0.0–1.0). Default: 0.3

Output Ports: - result (TEXT) — JSON object with segmentation results (mock data)

Result Schema: 

{
  "masks": [
    {
      "mask_index": 0,
      "bbox": [x1, y1, x2, y2],
      "score": 0.75,
      "area": 3000,
      "phrase_match": "brown wooden chair"
    }
  ],
  "count": 1,
  "query": "brown wooden chair"
}

3.5 Track Init (SAM2+, Mock)

Node Type: model_sam__track_init

STATUS: Mock backend — returns random placeholder data. Real implementation requires SAM2+ backend.

Initialize object tracking on a segmented mask. Subsequent frames can use Track Propagate to follow the object.

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image (initial frame) - mask_index (TEXT) — Mask index from a prior segmentation result (e.g. from Segment Point) - points (TEXT, optional) — JSON list of [x, y] points to help identify the object if mask is ambiguous

Output Ports: - result (TEXT) — JSON object with tracking initialization result (mock data)

Result Schema: 

{
  "track_id": "trk_a1b2c3d4",
  "status": "initialized",
  "initial_bbox": [100, 100, 200, 200]
}

3.6 Track Propagate (SAM2+, Mock)

Node Type: model_sam__track_propagate

STATUS: Mock backend — returns random placeholder data. Real implementation requires SAM2+ backend.

Propagate tracked objects to a new frame. Provide track IDs from Track Init and a new image, receive updated bboxes and confidence scores.

Input Ports: - image_b64 (TEXT) — Base64-encoded PNG or JPEG image (new frame) - track_ids (TEXT) — JSON list of track IDs from Track Init, e.g. ["trk_a1b2c3d4", "trk_e5f6g7h8"]

Output Ports: - result (TEXT) — JSON object with track propagation results (mock data)

Result Schema: 

{
  "tracks": {
    "trk_a1b2c3d4": {
      "bbox": [105, 105, 205, 205],
      "score": 0.92,
      "frame_count": 1,
      "status": "tracking"
    },
    "trk_e5f6g7h8": {
      "status": "lost",
      "score": 0.0
    }
  },
  "active_count": 1
}

4. Configuration

4.1 Environment Variables

Set these before the backend initializes:

# Select backend and model
export SAM_VERSION=sam2
export SAM_MODEL_CFG=default
export SAM_DEVICE=cuda:0

# Start the backend
python -c "from workspace.nodesets.sam import SamNodeSet; ..."

4.2 Model Checkpoints

Checkpoints are loaded from disk during initialize(). Ensure the path exists:

# SAM v1 checkpoint (included with segment_anything pip package)
data/habitat/checkpoints/sam/sam_vit_b.pth

# Custom path
export SAM_MODEL_CFG=/path/to/your/sam_model.pth

4.3 Node UI Configuration

The Auto Mask node exposes a slider for points_per_side in the canvas UI:

class SamAutoMaskTool(BaseCanvasNode):
    ui_config: ClassVar[NodeUIConfig] = NodeUIConfig(
        color="teal",
        config_fields=[
            ConfigField("points_per_side", "slider", label="Points/side",
                       default=32, min=8, max=64, step=8)
        ],
    )

Users can adjust grid density via the node's config panel on the canvas without editing JSON.

4.4 Port Serialization

All image and mask data flows as base64-encoded PNG strings over the wire:

# Image encode/decode
def _encode_image(rgb_array: np.ndarray) -> str:
    """RGB uint8 (H, W, 3) → base64 PNG"""
    img = Image.fromarray(rgb_array)
    buf = io.BytesIO()
    img.save(buf, format="PNG")
    return base64.b64encode(buf.getvalue()).decode("ascii")

def _decode_image(image_b64: str) -> np.ndarray:
    """base64 PNG → RGB uint8 (H, W, 3)"""
    buf = io.BytesIO(base64.b64decode(image_b64))
    img = Image.open(buf).convert("RGB")
    return np.array(img, dtype=np.uint8)

# Mask encode/decode (same pattern)
# Masks are (H, W) boolean, encoded as grayscale PNG (0 or 255)

5. Usage

5.1 Loading the NodeSet

The SAM NodeSet is discovered automatically via the component registry. To load it:

Canvas UI: 1. Go to the Manager tab 2. Under Local NodeSets, find "sam" 3. Click Load 4. All six SAM nodes appear in the sidebar

Backend API: 

curl -X POST http://localhost:8000/api/components/nodesets/sam/load

Python (direct): 

from workspace.nodesets.sam import SamNodeSet
import asyncio

async def main():
    nodeset = SamNodeSet()
    await nodeset.initialize()
    tools = nodeset.get_tools()
    print(f"Loaded {len(tools)} SAM tools")
    # ... use tools ...
    await nodeset.shutdown()

asyncio.run(main())

5.2 Example Workflow: Segment Object from Instruction

Goal: Agent receives instruction "go to the red door" and needs to find the door in the current view.

Workflow:

1. User provides image and instruction "red door"
   │
2. SAM: Auto Mask
   Input: image_b64
   Output: masks for all objects in scene
   │
3. Skill (external) matches "red door" to mask(es)
   │
4. Extract that mask's bounding box
   │
5. Use bounding box or points to refine with SAM: Segment Point
   │
6. Return final mask + bbox to navigation planner

Graph pseudocode: 

{
  "nodes": [
    {
      "id": "auto_mask",
      "node_type": "model_sam__auto_mask",
      "inputs": {
        "image_b64": "..."
      }
    },
    {
      "id": "refine_segment",
      "node_type": "model_sam__segment_point",
      "inputs": {
        "image_b64": "...",
        "points": "[[200, 150]]",
        "labels": "[1]"
      }
    }
  ],
  "wires": [
    {
      "from": "auto_mask.result",
      "to": "refine_segment.image_b64"
    }
  ]
}

5.3 Example Workflow: Track Landmark Across Video

Goal: Agent sees a lamp post and needs to track it across multiple frames to estimate motion/camera pose.

Workflow (requires SAM2+ backend):

Frame 1:
1. SAM: Auto Mask (or Segment Point)
   Output: mask_index=3 (lamp post)
   │
2. SAM: Track Init
   Input: image_b64, mask_index=3
   Output: track_id="trk_abc123"

Frame 2:
3. SAM: Track Propagate
   Input: new image_b64, track_ids=["trk_abc123"]
   Output: bbox=[150, 100, 200, 250], score=0.93

Frame 3:
4. SAM: Track Propagate (again)
   Input: newer image_b64, track_ids=["trk_abc123"]
   Output: bbox=[148, 98, 202, 252], score=0.91
   │
5. Estimator (external) computes motion from bboxes

Graph pseudocode: 

{
  "nodes": [
    {
      "id": "init_tracking",
      "node_type": "model_sam__track_init",
      "inputs": {
        "image_b64": "frame1.b64",
        "mask_index": 3
      }
    },
    {
      "id": "propagate",
      "node_type": "model_sam__track_propagate",
      "inputs": {
        "image_b64": "frame2.b64",
        "track_ids": "[\"trk_abc123\"]"
      }
    }
  ]
}

5.4 Error Handling

When a tool encounters an unsupported feature, it returns a descriptive error:

# SAM v1 backend does NOT support tracking
backend = Sam1Backend()
result = await backend.track_init(image_b64, mask_index=0)
# Returns:
# {
#   "error": "sam does not support tracking. Use sam2 or sam3."
# }

Check backend.supports_tracking and backend.supports_text_prompt before calling:

if not backend.supports_tracking:
    print("Tracking unavailable; use SAM2 or SAM3")
    # Fallback to point-based segmentation
