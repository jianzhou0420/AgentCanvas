Pre-commit Hooks

The commit-time check suite that keeps the repository clean, formatted, secret-free, and conventionally committed.

AgentCanvas uses pre-commit to run a fixed suite of checks on every commit. The single source of truth is .pre-commit-config.yaml at the repo root. Hooks run in the order hygiene → formatters → linters → security → commit-msg; formatters run before linters so lint errors reflect real issues, not whitespace.

§1Install (once after clone)

pip install pre-commit
pre-commit install
pre-commit install --hook-type commit-msg   # enable the commit-msg hook

The first pre-commit install wires the pre-commit git hook (runs on git commit over staged files). The second, --hook-type commit-msg, is required for the Conventional Commits check to fire on your commit message — without it, that hook silently never runs.

§2Run

Hooks run automatically on git commit over the staged files. To check the whole repo on demand (do this before pushing):

pre-commit run --all-files
Dirty-tree note pre-commit stashes unstaged changes while it runs. On a heavily-dirty tree that stash can conflict with hook auto-fixes; if that happens, verify the staged set directly with pre-commit run --files <paths> and commit that exact set. Never --no-verify past a real hook failure.

§4Notable config & exclusions

  • Large-file cap. check-added-large-files uses --maxkb=500. Curated large content is path-excluded: outputs/archive/ (compressed run traces), architect-site/reference/_archive/ (frozen reference), and assets/readme/ (README hero GIFs).
  • Vendored code is skipped. The whitespace / EOF / debug-statement / line-ending hooks exclude third_party/, workspace/nodesets/policy/policy_vla/(adapters|models|policies)/, and workspace/nodesets/env/env_detany3d/_vendor/ — upstream code we don't reformat.
  • JSONC tolerated. check-json excludes .vscode/ (JSON-with-comments by design) and the frontend package-lock.json.
  • Secret scanning. Before going public, also scan the full history once: 
    gitleaks detect --source . --log-opts="--all"
  • Conventional Commits runs only at the commit-msg stage, so it needs the separate --hook-type commit-msg install above. Commit subjects follow type(scope): description (lowercase, imperative, ≤72 chars) — see Contributing § PR format.
