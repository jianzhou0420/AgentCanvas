Repository Governance

The meta-documents that govern the AgentCanvas repository — how it is versioned, contributed to, licensed, and checked at commit time.

This section gathers the repository's governance documents — how AgentCanvas is versioned, contributed to, licensed, and checked at commit time. The version number is driven by API stability and is kept independent of paper milestones; that decoupling, plus security and citation guidance, are spelled out in the Versioning Policy.

§2The governance model in one line

A small, tightly-held core (evolved through ADRs) plus a large content ecosystem — nodesets and graphs contributed by PR into workspace/ and credited on the Credits board. Most contributions are content (light review); the framework itself is contributed via reviewed PRs, with a Discussion first for anything big. The full breakdown is in Contributing; the paradigm-level meaning of each major version is in Major Versions.
