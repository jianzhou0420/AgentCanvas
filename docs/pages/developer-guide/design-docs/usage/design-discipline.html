Design Discipline

  1. A share-mode (parallelism="shared") server must not hold nodeset-level state.

    Why:

    • Design philosophy — only stateless things may run in shared mode.
    • Unlike replicated mode (one state per worker), shared mode uses a single shared instance, so there is no per-worker state to hold.
    • Might be supported later; not supported now.
  2. State must be passive data, not a live object.

    Why: so the state can be accessed by other processes. We transfer data across servers, and a live Python object can't be transferred across processes.

  3. An env nodeset must follow the env nodeset template.
