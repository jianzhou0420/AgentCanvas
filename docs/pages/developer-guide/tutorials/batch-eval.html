Running a Parallel Batch Eval

Evaluate one graph against many episodes in parallel: K env subprocesses step concurrently while a single shared GPU policy server batches their inference calls. End-to-end walkthrough — from clicking Start on the Eval page to opting a custom GPU node into the batched-inference path.

This is a tutorial. For how the worker pool is wired or why the policy server is shared instead of replicated, read Design — Batch Eval System (and ADR-eval-002 / ADR-eval-004). To drive this whole flow from a script or coding agent instead of the UI, see Driving the Backend from a Coding Agent.

1. Prerequisites

  • Backend running on :8000 (your own session) or under /experiment:run (allocated $BACKEND_URL); canvas UI on :5173.
  • A graph that uses an env nodeset — e.g. workspace/graphs/cma_baseline.json for env_habitat + policy_cma.
  • The env's dataset already in place (Habitat-Sim split downloaded, policy checkpoint under data/policy/, etc.).

2. First Parallel Run

  1. Open the Eval page in the UI.
  2. Pick your graph. Set Episode Count to e.g. 50.
  3. Set Worker Count to 4. Leave the rest at defaults.
  4. Click Start.

What you should see:

  • The progress strip splits into 4 rows, one per worker.
  • Episodes complete in arbitrary order; the final summary.json sorts them back by episode_index.
  • VRAM stays roughly flat as you raise worker_count — one policy server, K env subprocesses.

If wall-clock doesn't drop ≈linearly, your bottleneck is the GPU forward, not the env step — § 4 covers turning that knob.

3. Tune the Per-Step Budget

Each episode is capped at max_steps × per_step_budget_sec. A single slow step aborts that episode with error="timeout after Xs" — the worker resumes the queue, so one stuck episode never blocks the pool.

Resolution order, highest precedence first:

  1. EvalConfig.per_step_budget_sec from the eval form.
  2. The env nodeset's default_per_step_budget_sec class var.
  3. BaseNodeSet.default_per_step_budget_sec = 5.0 s.
NodesetDefaultRationale
env_habitat2.0 sHabitat-Sim step is sub-second; 2 s leaves GC headroom.
Framework fallback5.0 sConservative for unknown nodesets.
LLM-heavy graphs (MapGPT, NavGPT)30.0 sNetwork round-trip + provider queueing.

Override the budget when:

  • Your graph runs an LLM call inside the loop and per-call latency varies a lot (rate limits, long prompts).
  • You're profiling and want a tight budget so regressions abort fast.

4. Batch a Custom GPU Node

Heavy GPU nodes (policy networks, local VLMs) should multiplex K parallel worker calls into one forward pass instead of being duplicated K times in VRAM. Three changes turn a normal node into a batched one.

4.1 Declare the opt-in

class PolicyForwardTool(BaseCanvasNode):
    node_type = "policy_cma__forward"
    batched: ClassVar[bool] = True
    batch_dim: ClassVar[str] = "raw_obs"   # which input port carries the per-sample slot
    input_ports = [
        PortDef("raw_obs",   "TEXT", "Raw observation"),
        PortDef("hidden_in", "ANY",  "Prior RNN state", optional=True),
    ]
    output_ports = [
        PortDef("action",     "ACTION", "Predicted action"),
        PortDef("hidden_out", "ANY",    "RNN state for next step"),
    ]

Scanned at registration: if batched=True then batch_dim must name a real input port, or register_node() raises ValueError.

4.2 Move per-call state onto the wire

The batched server is pure-functional — it holds no per-worker state. Anything that used to live on ctx (RNN hidden states, prior action, episode mask) becomes an explicit input port and an output port, fed back through the loop's IterIn / IterOut:

policy.hidden_out ──► iterOut.hidden ──► (next iter) ──► iterIn.hidden ──► policy.hidden_in

On iteration 0 the IterIn carries None (no upstream source on hidden from init); the policy initialises zeros internally.

4.3 Implement the batched forward()

The framework hands all K samples under a marker key; the node returns all K outputs under another:

from app.server.batched_inference import OUTPUTS_KEY, SAMPLES_KEY

async def forward(self, inputs, ctx):
    samples = inputs.get(SAMPLES_KEY)
    is_batched = isinstance(samples, list)
    if not is_batched:
        samples = [inputs]                 # local mode: degenerate batch of 1

    # ... stack along batch_dim, call the model once, split outputs ...

    if is_batched:
        return {OUTPUTS_KEY: outputs}      # list, one entry per sample
    return outputs[0]

At worker_count=1 the queue still flushes after flush_timeout_ms (default 50 ms) with batch size 1 — adds a 50 ms tail per inference call but keeps the contract identical across all K.

4.4 Pitfalls

  • Don't put state on ctx in a batched node. The handler is invoked once per flush, so K workers share one ctx. Use ports.
  • Don't read module-level mutables from inside forward() — same shared-state trap, one level removed.
  • Non-batched local nodes (e.g. basic_agent history accumulators) are unaffected because they're never multiplexed across workers.

5. Debug a Failing Episode

Every EpisodeResult carries a worker_id (0..N-1) and owns a self-contained log dir under episodes/ep{NNNN}/ (ADR-eval-004) — reaching one episode's firings is a path, not a JSONL filter.

5.1 Inspect one episode's log

# Episode 47's firings (file IS the episode):
jq -c '.' outputs/eval_runs/<run_id>/episodes/ep0047/log.jsonl

# Worker assignment across all episodes in the run:
for ep in outputs/eval_runs/<run_id>/episodes/ep*/episode.json; do
  jq -r '"\(.episode_index)\tworker=\(.worker_id)\tstatus=\(.status)"' "$ep"
done

5.2 Reproduce single-tenant

If episode 47 errored on worker 2, narrow the run to just it:

worker_count        = 1
episode_count       = 1
start_episode_index = 47

Same code path through _run_one_episode, no worker interleaving — easy to attach a debugger and read the logs linearly.

5.3 Watch progress live

The Eval UI shows N progress strips when worker_count > 1. The per_worker_status field on each eval_progress WebSocket frame also lists every worker's currently-running episode index — useful when you're driving the run from a script.
